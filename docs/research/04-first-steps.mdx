## Repository Setup

### SiafuDB (`nyuchi/siafudb`)

1. Push scaffolded codebase (crates/, Cargo.toml, README.md)
2. Add LICENSE (Apache 2.0), CONTRIBUTING.md, CODE_OF_CONDUCT.md, THIRD_PARTY_NOTICES
3. Create `docs/` for Mintlify content (hosted at `siafu.nyuchi.com`)
4. Move research `.mdx` files into `docs/research/`
5. Enable GitHub Discussions, Issues, branch protection on `main`
6. Topics: `graph-database`, `embedded-database`, `rust`, `edge-computing`, `web3`, `sync-protocol`, `apache-2`, `open-source`, `africa`

### NTL (`nyuchi/ntl`) — Separate Actions

NTL has its own first-steps document in its repo. SiafuDB-relevant NTL actions:

- Create `ntl-signal` crate with Node, Signal, Synapse types
- Implement channel transport layer (in-process foundation)
- Define the GSPN signal interface that SiafuDB will integrate with
