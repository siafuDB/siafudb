## Stability

The specifications versioned in this section are **normative**. Breaking changes require a major-version bump and a migration path. Non-breaking clarifications can land under the same version.

## Research vs. specification

The [Research](/research/overview) section carries the reasoning — why the model is shaped the way it is. The Specification section carries the **rules** — exactly what an implementation must do. If the two conflict, the specification wins; file an issue to correct the research document.
