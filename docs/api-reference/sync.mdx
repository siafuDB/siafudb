The sync layer lives in `siafudb-sync` and is re-exported under `siafudb::sync`.

## GSPA

```rust
use siafudb::sync::Gspa;

let gspa = Gspa::http()
    .from(&db)
    .to("https://platform.example.com/sync")
    .transform("StripPII")
    .auth_bearer(token)
    .build()?;

gspa.start().await?;
```

Transport variants: `Gspa::http()`, `Gspa::grpc()`, `Gspa::websocket()`, `Gspa::kafka()`, `Gspa::local_tcp()`, `Gspa::in_process()`.

See [GSPA](/sync/gspa).

## GSPN

```rust
use siafudb::sync::Gspn;

let gspn = Gspn::new()
    .from(&db)
    .to_synapse("synapse://platform-ingress")
    .build()?;

gspn.start().await?;
```

See [GSPN](/sync/gspn).

## CDC hook

If you need to observe mutations without shipping them, subscribe directly:

```rust
let mut cdc = db.cdc_subscribe();
while let Some(mutation) = cdc.recv().await {
    tracing::info!(?mutation, "mutation committed");
}
```

## Transforms

Built-in transforms:

| Transform | Effect |
|-----------|--------|
| `StripPII` | Removes properties marked `@pii` in the schema |
| `FilterByLabel` | Drops mutations whose target label doesn't match |
| `Anonymise` | Replaces identities with stable pseudonyms |
| `Aggregate` | Rolls up events into summary vertices |

Custom transforms implement the `Transform` trait:

```rust
impl Transform for MyTransform {
    fn apply(&self, m: Mutation) -> Option<Mutation> { ... }
}
```

## Conflict strategy

Set per-schema or per-fragment:

```rust
db.set_conflict_strategy(Label::new("Message"), Strategy::MergeProperties)?;
```

See [Conflict Resolution](/spec/conflict-resolution).
